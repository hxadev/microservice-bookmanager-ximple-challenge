## Description
Solution to the challenge proposed by the company "Ximple" to level up the knowledge of a senior backend developer. \
The main focus of the solution was on generating APIs using the OpenAPI standard and the "openapi-generator" tool, which is included in the Spring Boot project. The code is generated based on the specification provided in the openapi.yaml file located in the /src/main/resources folder. This file follows the industry-standard API design.

### Project Information
- Java Version: 21
- Spring Boot Version: 3.0
- Spring Security: 6.0
- OpenAPI Spec: 3.0.3

## Paths

### List all Books
- **Endpoint**: `/books`
- **Method**: GET
- **Summary**: List all the books
- **Operation ID**: listBooks
- **Responses**:
    - 200: A list of books
    - 400: Invalid request
    - 401: Unauthorized
    - 404: Resource not found
    - 500: Internal Server Error

### Create a new Book
- **Endpoint**: `/books`
- **Method**: POST
- **Summary**: Create a new book
- **Operation ID**: createBooks
- **Request Body**:
    - Required: true
    - Content Type: application/json
    - Schema: [Book](#schemas)
- **Responses**:
    - 201: Book Created
    - 400: Invalid request
    - 401: Unauthorized
    - 500: Internal Server Error

### Get Info for a Specific Book
- **Endpoint**: `/books/{bookId}`
- **Method**: GET
- **Summary**: Info for a specific book
- **Operation ID**: showBookById
- **Parameters**:
    - name: bookId
      in: path
      description: The book id
      required: true
      schema:
      type: integer
- **Responses**:
    - 200: The required book
    - 400: Invalid request
    - 401: Unauthorized
    - 404: Resource not found
    - 500: Internal Server Error

### Delete all Books
- **Endpoint**: `/books/{bookId}`
- **Method**: DELETE
- **Summary**: Delete all books
- **Operation ID**: deleteBooks
- **Parameters**:
    - name: bookId
      in: path
      description: The book id that needs to be deleted
      required: true
      schema:
      type: integer
- **Responses**:
    - 204: The resource was deleted successfully
    - 400: Invalid book id supplied
    - 401: Unauthorized
    - 404: The book was not found

### Authenticate User and Retrieve JWT
- **Endpoint**: `/users`
- **Method**: POST
- **Summary**: Endpoint to authenticate the user and retrieve JWT
- **Operation ID**: authUser
- **Request Body**:
    - Content Type: application/x-www-form-urlencoded
    - Schema:
        - user (string, example: "admin")
        - password (string, example: "1234")
- **Responses**:
    - 200: Authentication successfully
    - 401: Authentication failed
